# User Flow: PM Feedback Analyzer

## Main user
A product manager who has collected raw user feedback and wants to turn it into actionable insight.

## Primary flow
1. User lands on the product page
2. User understands the value proposition
3. User pastes raw feedback into the input box
4. User clicks "Analyze Feedback"
5. User receives a structured output
6. User reviews:
   - key themes
   - main pain points
   - repeated feature requests
   - suggested priorities
7. User copies the output or uses it for product planning

## Expected outcome
The user moves from messy feedback to clear product insight in a fast and simple way.

## Key value
- saves time
- reduces manual review effort
- makes feedback easier to act on
- helps PMs prioritize better
