# MVP Prototype Plan v1 — SignalForge

## Prototype goal
Create a believable and coherent prototype that demonstrates the SignalForge workflow from feedback input to insights, report, PRD, and task output.

The prototype does not need to implement real backend logic in v1.
It needs to clearly communicate the product value, workflow, and output quality.

## Prototype principle
Prototype for clarity first, not for technical completeness.

## What the prototype must prove
- SignalForge is a workflow, not just a single AI tool
- The user can move from input to structured outputs
- Each step feels connected to the previous step
- The product feels enterprise-ready and structured
- The outputs feel useful for real product work

## Prototype type
Interactive front-end prototype with realistic sample content

## Recommended prototype approach
Use a project-based workflow with:
- one Projects Home screen
- one project workspace shell
- sequential workflow screens
- realistic but pre-written outputs

## Prototype scope

### Screens to build first
1. Landing Page
2. Projects Home
3. Create Project
4. Input Workspace
5. Insights View
6. Report View
7. PRD Studio
8. Task Builder

## Prototype build order

### Phase 1
- Landing Page
- Projects Home
- Project Workspace shell
- Input Workspace
- Insights View

### Phase 2
- Report View
- PRD Studio
- Task Builder

### Phase 3
- polish
- loading states
- empty states
- success feedback
- copy/export interactions

## Prototype interaction model

### Real interactions
- create a project
- move between workflow steps
- paste or edit input text
- click generate buttons
- navigate to next step
- see outputs appear

### Fake or simulated interactions
- actual AI generation
- actual CSV parsing
- true backend saving
- true export generation
- real account system

## Content strategy
The prototype should use one consistent demo scenario across all screens.

## Recommended demo scenario
Project: Match Experience Improvement

Goal:
Turn recent user feedback into structured insights, a PRD draft, and execution-ready tasks.

## Demo input sample
- The app is useful but finding past sessions is confusing.
- I want an easier way to reschedule matches.
- Notifications are helpful, but there are too many of them.
- It takes too many steps to join a match.
- I would love better filters for searching players.
- Match reminders sometimes come too late.

## Demo outputs to prepare
- insight output
- report output
- PRD output
- task output

## Prototype quality bar
The prototype should feel:
- coherent
- structured
- realistic
- professional
- presentation-ready

It does not need to feel technically complete.

## Key risk to avoid
Do not try to prototype every possible feature.
The first prototype should show the core workflow clearly.
