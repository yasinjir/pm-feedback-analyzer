# Feature Priority v1 — SignalForge

## Tier 1 — Must have
These features are required for the MVP to make sense.

- Create project
- Paste raw feedback
- Upload CSV
- Generate insights
- Generate report
- Generate PRD draft
- Generate task draft

## Tier 2 — Should have
These features improve the product, but the MVP can exist without them.

- Editable PRD blocks
- Copy to clipboard
- Re-run analysis
- Better formatting of outputs
- Simple export-ready views

## Tier 3 — Later
These features should come after the MVP is validated.

- Form builder
- Survey links
- SMS ingestion
- Collaboration
- Integrations with PM tools
- Permissions
- Workflow automation
- Advanced dashboards

## Priority principle
The MVP should prioritize the shortest path from feedback input to structured output.

## Design principle
SignalForge v1 should behave like one coherent workflow, not like four disconnected tools.
