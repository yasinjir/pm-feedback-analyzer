# SignalForge Screen Map v1

## 1. Landing Page
Purpose: explain the product vision and the current public module.

## 2. Create Project
Purpose: define the project context before analysis.

### Fields
- project name
- feature or initiative name
- optional context
- analysis goal

## 3. Input Workspace
Purpose: collect and import feedback.

### Input types
- raw text
- pasted notes
- table-like input
- CSV upload

## 4. Insights View
Purpose: turn raw input into structured insight.

### Sections
- themes
- pain points
- feature requests
- repeated signals
- priority cues
- summary insight

## 5. Report View
Purpose: create an executive summary of findings.

### Sections
- executive summary
- key findings
- top problems
- suggested opportunities
- recommended focus

## 6. PRD Studio
Purpose: convert validated insights into a standardized PRD draft.

### Sections
- problem statement
- goals
- target users
- scope
- non-goals
- success metrics
- risks
- open questions

## 7. Task Builder
Purpose: convert PRD output into execution-ready work.

### Sections
- work buckets
- tasks
- user stories
- acceptance criteria
