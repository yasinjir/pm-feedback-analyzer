# Screen Definition v1 — SignalForge

## 1. Landing Page

### Purpose
Explain the SignalForge vision and help the user understand the workflow from feedback to PRD to execution.

### Primary user goal
Understand the product and start the workflow.

### Main sections
- hero section
- product architecture
- current live module
- workflow overview
- strategic value
- final CTA

### Primary action
Start the workflow

### Secondary action
Review current module

### Required information
- product name
- one-line positioning
- key workflow
- current module focus
- high-level proof of value

### Key UX requirement
The user should understand what SignalForge does in less than 30 seconds.

---

## 2. Create Project

### Purpose
Define the working context before feedback analysis begins.

### Primary user goal
Set up a project with enough context for meaningful output generation.

### Fields
- project name
- feature or initiative name
- optional background context
- analysis goal

### Primary action
Create project

### Secondary action
Cancel or go back

### Validation rules
- project name is required
- analysis goal is required

### Key UX requirement
The screen should feel lightweight and fast, not like a long setup form.

---

## 3. Input Workspace

### Purpose
Allow the user to provide raw product signals and feedback.

### Primary user goal
Add enough input to start insight generation.

### Supported inputs in v1
- raw text
- note-like content
- table-like content
- CSV upload

### Main sections
- input method area
- raw input editor
- file upload area
- input guidance
- analysis CTA

### Primary action
Generate insights

### Secondary action
Replace input / edit input

### Key UX requirement
The user should clearly understand what kinds of input are accepted.

---

## 4. Insights View

### Purpose
Convert raw input into structured product insight.

### Primary user goal
Review structured output and decide whether it is usable.

### Main sections
- themes
- pain points
- feature requests
- repeated signals
- priority cues
- summary insight

### Primary action
Generate report

### Secondary action
Refine input / re-run analysis

### Key UX requirement
The screen should feel structured and decision-oriented, not just like an AI summary.

---

## 5. Report View

### Purpose
Present an executive-friendly summary of the insight output.

### Primary user goal
Get a concise and shareable view of findings.

### Main sections
- executive summary
- key findings
- top problems
- suggested opportunities
- recommended focus

### Primary action
Generate PRD draft

### Secondary action
Go back to insights

### Key UX requirement
The report should be readable by stakeholders beyond the PM.

---

## 6. PRD Studio

### Purpose
Turn insight into a standardized product requirements document draft.

### Primary user goal
Get a usable draft PRD that can be reviewed and edited.

### Main sections
- problem statement
- goals
- target users
- scope
- non-goals
- success metrics
- risks
- open questions

### Primary action
Generate task draft

### Secondary action
Edit PRD

### Key UX requirement
The output should feel like a real product document, not a generic template.

---

## 7. Task Builder

### Purpose
Convert the PRD draft into execution-ready delivery input.

### Primary user goal
Get structured work items that can be used for implementation planning.

### Main sections
- work buckets
- task list
- user stories
- acceptance criteria

### Primary action
Copy or export tasks

### Secondary action
Return to PRD

### Key UX requirement
The output should feel useful for handoff to engineering or delivery teams.

---

## 8. Export / Copy Layer

### Purpose
Let the user take outputs out of SignalForge for planning and execution.

### Primary user goal
Reuse outputs outside the product.

### Supported outputs in v1
- copied text
- export-ready text blocks

### Primary action
Copy output

### Secondary action
Return to workflow

### Key UX requirement
Export should be simple and low-friction.
