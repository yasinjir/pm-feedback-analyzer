# Demo Scope v1 — SignalForge

## Demo objective
Show the shortest convincing path from feedback input to product action.

## Demo storyline
A product manager starts with scattered user feedback and uses SignalForge to:
1. define a project
2. add product inputs
3. generate structured insights
4. review a summary report
5. generate a PRD draft
6. generate a task draft

## Demo screens required

### 1. Landing Page
Purpose:
Introduce SignalForge and explain the workflow.

### 2. Projects Home
Purpose:
Show that the product is project-based.

### 3. Create Project
Purpose:
Set context for the workflow.

### 4. Input Workspace
Purpose:
Show how raw product signals enter the system.

### 5. Insights View
Purpose:
Show the core transformation from input to insight.

### 6. Report View
Purpose:
Show stakeholder-friendly output.

### 7. PRD Studio
Purpose:
Show how insight becomes a structured product document.

### 8. Task Builder
Purpose:
Show how the PRD becomes execution-ready work.

## Demo actions required
- Create project
- Add or review input
- Generate insights
- Generate report
- Generate PRD
- Generate tasks
- Copy output

## Demo states required
- default state
- loading state
- success state
- empty state for at least one screen

## What can be static
- generated content
- report details
- PRD details
- task structure

## What should feel interactive
- navigation
- input editing
- generate buttons
- movement between steps
- copy action feedback

## Demo success criteria
A viewer should understand:
- what SignalForge does
- why the workflow matters
- what the current MVP includes
- how the product moves beyond simple summarization

## Demo scope rule
If a feature does not strengthen the core workflow demo, it should not be included in the first prototype.
