# User Flow v1 — SignalForge

## Primary user
Product Manager

## Primary goal
Move from raw feedback to structured insights, a PRD draft, and task drafts in one coherent workflow.

## Main flow

### Step 1: Land on SignalForge
The user understands the product promise and chooses to start a new workflow.

### Step 2: Create a project
The user enters:
- project name
- feature or initiative name
- optional background context
- analysis goal

### Step 3: Add product inputs
The user provides raw input by:
- pasting feedback text
- pasting note-like content
- pasting table-like content
- uploading CSV

### Step 4: Generate insights
SignalForge processes the input and generates:
- themes
- pain points
- feature requests
- repeated signals
- priority cues
- summary insight

### Step 5: Review insight output
The user reviews the structured output and decides whether it is good enough to continue.

### Step 6: Generate report
SignalForge creates a management-friendly report view with:
- executive summary
- key findings
- top problems
- suggested opportunities
- recommended focus

### Step 7: Generate PRD draft
The user asks SignalForge to convert the insights into a standardized PRD draft.

The PRD draft includes:
- problem statement
- goals
- target users
- scope
- non-goals
- success metrics
- risks
- open questions

### Step 8: Generate task draft
The user asks SignalForge to convert the PRD into execution-ready work.

The task draft includes:
- work buckets
- task list
- user stories
- acceptance criteria

### Step 9: Export or copy outputs
The user copies or exports the generated outputs for planning, alignment, or delivery.

## Short navigation model
Landing → Create Project → Inputs → Insights → Report → PRD → Tasks

## Core principle
The user should always feel that each step is a natural continuation of the previous one.
